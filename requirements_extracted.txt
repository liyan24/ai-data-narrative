ai-data-narrative 系统 - 需求文档
1. 项目概述
1.1 目的
ai-data-narrative 是一个 AI 驱动的工作流系统，能够自动将各种类型的数据转化为引人注目的数据叙事。它实现了 Cole Nussbaumer Knaflic 所著《Storytelling with Data》中的六步方法论，并提供了一个使用 DeepEval 启发的指标、支持多 LLM 评估的五维质量评估框架。
1.2 范围
该系统覆盖完整的生命周期：
输入：用户请求 + 元数据 + 数据项 + 可选背景
处理：6 步 AI 工作流（背景 → 分析 → 可视化 → 聚焦 → 设计 → 叙事）
输出：最终叙事报告 + 可视化图表 + 生成的代码
评估：5 指标质量评估，支持多 LLM 评审
1.3 目标用户
需要向利益相关方传达数据分析结果的数据科学家
需要从数据中生成结构化叙事报告的研究人员

2. 功能需求
2.1 核心工作流（FR-CW-001 至 FR-CW-006）
FR-CW-001：第一步 - 理解背景
描述：分析用户请求和背景，建立叙事策略
输入：user_request（字符串）、background（字符串）、audience（字符串）、data（json）、data_description（json）
输出：
背景简报（受众画像、沟通策略）
大创意（一句话核心信息）
三分钟故事（电梯演讲）
7 页故事板
智能体：ContextAnalysisAgent
框架：Who/What/How、探索性 vs 解释性分析
LLM 提示模板：必须生成包含所有输出字段的结构化 JSON
优先级：关键
FR-CW-002：第二步 - 数据分析
描述：计算全面的数据指标并识别关键发现
输入：关键发现数据、第一步的分析计划
输出：
关键发现数据
Python 可视化代码
智能体：DataAnalysisAgent
技能：DataAnalysisSkill
代码生成：生成可执行的 Python 代码
优先级：关键
FR-CW-003：第三步 - 选择可视化并消除杂乱
描述：选择适当的可视化类型并应用去杂乱原则
输入：第二步的分析结果
输出：
可视化规格（图表类型、数据映射、尺寸）
去杂乱计划（移除边框、网格线、标签）
前后对比可视化图
格式塔原则应用日志
智能体：VisualizationDesignAgent（去杂乱模式）
原则：
移除：边框、网格线、过多标签、背景填充、3D 效果
应用：接近性、相似性、闭合性、连续性、连接性
低优先级图表：饼图、环形图、3D 图表、双 Y 轴
优先级：高
FR-CW-004：第四步 - 聚焦注意力
描述：使用先于注意的属性引导观众视线
输入：第三步的去杂乱可视化图
输出：
注意力聚焦的可视化图
先于注意属性映射（颜色、大小、位置分配）
视觉层次规格
“目光被吸引” 测试结果
智能体：VisualizationDesignAgent（聚焦模式）
先于注意属性：
颜色：最多 3-4 种，金色（#FFD700）用于关键元素
大小：与重要性成正比
位置：关键元素放在黄金位置
形状：关键类别使用独特形状
高亮限制：最多 10% 的视觉元素被高亮
优先级：高
FR-CW-005：第五步 - 像设计师一样思考
描述：应用设计师思维进行最终润色
输入：第四步的注意力聚焦可视化图
输出：
生产级可视化图
功能可见性规格
无障碍检查清单结果
美学一致性验证
智能体：VisualizationDesignAgent（设计模式）
原则：
功能可见性：交互元素显而易见
无障碍：色盲友好配色
美学：对齐、留白、专业外观
配色方案：
主色：#2E5C8A（深蓝）
辅色：#D9534F（红色）
强调色：#FFD700（金色）
背景：#FFFFFF（白色）
文字：#333333（深灰）
优先级：高
FR-CW-006：第六步 - 讲述故事
描述：构建最终的三幕叙事并编译报告
输入：所有前序步骤的输出
输出：
三幕叙事（铺垫、冲突、解决）
水平逻辑（仅标题就能讲故事）
可重复的精炼语句
内嵌可视化图表的最终 Markdown 报告
智能体：StorytellingAgent
框架：三幕结构、Bing-Bang-Bongo、水平/垂直逻辑
优先级：关键
2.2 质量评估（FR-QA-001 至 FR-QA-007）
FR-QA-001：信息完整性（IC）
描述：报告必须涵盖背景、方法、结论等关键部分
评分：每类 0.0-1.0，取平均得最终分
方法： LLM 判断
优先级：关键
FR-QA-002：事实准确性（FA）
描述：所有数值声明必须与数据一致
验证流程：
从叙事中提取所有数值声明（正则 + NLP）
从图中计算真实指标
通过关键词分析将声明与指标匹配
计算偏差：|声明 - 真相| / 真相
分类：
正确：偏差 ≤ 5%
近似：偏差 5-20%
错误：偏差 > 20%
评分：正确数 / 可验证总数
优先级：关键
FR-QA-003：叙事连贯性（NC）
描述：逻辑结构必须清晰且内部一致
评估维度：
结构（0.0-1.0）：开头、中间、结尾、过渡
证据支持（0.0-1.0）：声明有数据支撑
逻辑递进（0.0-1.0）：每节建立在前一节之上
内部一致性（0.0-1.0）：无矛盾
流畅性与连接性（0.0-1.0）：自然节奏、路标
自动检查：
是否有开头/中间/结尾标记
证据标记计数
逻辑连接词计数
矛盾检测
评分：5 个维度的平均值
优先级：关键
FR-QA-004：可理解性（CP）
描述：非专业读者必须能理解关键发现
评估维度：
术语处理（0.0-1.0）：首次使用时定义术语
句子复杂度（0.0-1.0）：目标每句 15-20 词
概念解释（0.0-1.0）：类比、直观解释
可视化可及性（0.0-1.0）：清晰标签、图例
假设知识（0.0-1.0）：先决条件已说明
结构辅助（0.0-1.0）：标题、摘要、高亮
评分：6 个维度的平均值
优先级：高
FR-QA-005：结构保真度（SF）
描述：叙事必须忠实反映数据真实情况
评分：6 个维度的平均值
优先级：关键
FR-QA-006：多 LLM 评估
描述：多个独立 LLM 评审员评估每个指标
支持的后端：OpenAI GPT-4、Anthropic Claude-3、Ollama（本地）
配置：
主评审：GPT-4 + Claude-3-Opus（权重 2.0）
副评审：GPT-3.5 + Claude-3-Sonnet（权重 1.0）
本地评审：通过 Ollama 的 Llama-3（权重 0.5）
聚合方法：
mean：简单平均
consensus：中位数（抗异常值）
weighted：按评审可靠性加权
一致性分析：
high：蟽 ≤ 0.15
medium：蟽 ≤ 0.30
low：蟽 > 0.30
降级：单 LLM 下系统仍可工作（降级模式）
优先级：关键
FR-QA-007：评估报告生成
描述：将所有指标结果编译为综合报告
输出格式：
Markdown（人类可读的表格）
JSON（机器可解析，用于 CI/CD 集成）
报告内容：
总分（加权平均）和等级（A+ 到 F）
每项指标：分数、标准差、一致性级别、各评审员分数
关键优势与劣势
改进建议
评审信息和聚合方法
等级标准：
A+ ≥ 0.95、A ≥ 0.90、A- ≥ 0.85
B+ ≥ 0.80、B ≥ 0.75、B- ≥ 0.70
C+ ≥ 0.65、C ≥ 0.60、C- ≥ 0.55
D ≥ 0.50、F < 0.50
通过阈值：默认 0.60（可配置）
优先级：高
2.3 智能体系统（FR-AS-001 至 FR-AS-006）
FR-AS-001：智能体基类
必须支持：init → plan → execute → review → report 生命周期
计划阶段：LLM 生成结构化执行计划（JSON）
执行阶段：按计划运行代码/调用技能
评审阶段：通过 LLM 自检结果质量
报告阶段：将发现编译为 AgentOutput
每个智能体拥有：name、role、system_prompt、skills 注册表
优先级：关键
FR-AS-002：ContextAnalysisAgent
角色：背景与策略分析师
实现《Storytelling with Data》第一课
输出：context_brief、big_idea、three_minute_story、storyboard
使用 Who/What/How 框架
优先级：关键
FR-AS-003：DataAnalysisAgent
角色：网络科学家
实现第二步（第二课）
生成 Python 可视化代码
使用 NetworkAnalysisSkill
优先级：关键
FR-AS-004：VisualizationDesignAgent
角色：数据可视化设计师
实现第三至五步（第三至五课）
三种模式：declutter → focus → design
每种模式输出不同的可视化图集
使用 DataVisualizationSkill
优先级：高
FR-AS-005：StorytellingAgent
角色：数据叙事师
实现第六步（第六至七课）
从所有前序输出编译三幕叙事
生成最终 Markdown 报告
使用 DataStorytellingSkill
优先级：关键
FR-AS-006：CodeReviewAgent
角色：质量保证工程师
检查：语法、安全、风格、可视化最佳实践
多层评审：自动 + LLM 驱动
优先级：高
2.4 技能插件系统（FR-SP-001 至 FR-SP-004）
FR-SP-001：技能基类
三阶段设计：plan() → validate_plan() → execute()
plan()：LLM 从上下文生成执行计划
validate_plan()：执行前检查可行性
execute()：运行 Python 代码执行技能
代码生成：LLM 编写任务专用 Python
代码提取：从 Markdown 代码块中解析
优先级：关键
FR-SP-002：DataAnalysisSkill
能力：指标计算、中心性、社区、结构
快速辅助函数
生成分析 Python 代码
优先级：关键
FR-SP-003：DataVisualizationSkill
基于数据关系选择图表类型
代码模板：network_graph、horizontal_bar、vertical_bar、line_graph、scatter、simple_text等
配色方案管理
去杂乱和聚焦计划
优先级：高
FR-SP-004：DataStorytellingSkill
大创意构思
三幕叙事构建
故事板创建（默认 7 页）
水平逻辑检查
Bing-Bang-Bongo 框架
优先级：高
2.5 待办管理（FR-TM-001 至 FR-TM-005）
FR-TM-001：任务生命周期
状态：PENDING → IN_PROGRESS → COMPLETED / FAILED / SKIPPED
转换：start_task()、complete_task()、fail_task()、skip_task()、retry_task()
优先级：关键
FR-TM-002：任务依赖
任务可依赖其他任务
自动阻塞：任务在依赖完成前不能开始
自动解除阻塞：当依赖完成时
优先级：关键
FR-TM-003：进度追踪
get_progress()：百分比、各状态计数
get_ready_tasks()：所有可开始的任务
get_next_task()：优先级最高的就绪任务
故事板导出：可视化面板展示
优先级：高
FR-TM-004：导出
to_markdown()：Markdown 格式的任务面板
to_storyboard()：便利贴风格的视觉面板
save()/load()：JSON 持久化
优先级：中
FR-TM-005：智能体集成
每个智能体创建/更新自己的任务
智能体运行时自动创建任务
自动追踪状态转换
优先级：高
2.6 代码执行引擎（FR-CE-001 至 FR-CE-005）
FR-CE-001：语法验证
执行前基于 AST 的 Python 语法检查
带行号的错误报告
优先级：关键
FR-CE-002：安全扫描
阻止导入：os.system、subprocess、eval、exec、socket、urllib、pickle、ctypes
阻止函数：eval()、exec()、compile()、import()
违规报告带严重级别分类
优先级：关键
FR-CE-003：沙箱执行
在受限命名空间中执行
捕获 stdout/stderr
上下文变量注入
超时保护（默认 120 秒）
优先级：关键
FR-CE-004：输出发现
自动发现生成的文件（PNG、CSV、JSON、HTML、MD）
追踪输出文件路径
优先级：高
FR-CE-005：重试逻辑
失败时自动重试（可配置最大重试次数）
重试间 LLM 驱动的代码修复
指数退避
优先级：中
